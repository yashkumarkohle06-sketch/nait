01

What is Cybersecurity — CIA Triad

Definition

Cybersecurity is the practice of protecting systems, networks, programs, and data from digital attacks, unauthorized access, damage, or theft. It ensures that information remains safe and systems remain functional.

The need for cybersecurity arises because modern systems store sensitive data — health records, financial info, personal details — and attackers constantly try to steal, damage, or disrupt this data.

The CIA Triad — 3 Core Goals of Security

Every cybersecurity concept revolves around one or more of these three pillars:

🔒 Confidentiality

Only authorized people can access data. Encryption ensures outsiders cannot read sensitive information even if they intercept it.

✅ Integrity

Data is accurate and has not been tampered with. Hashing and digital signatures verify that data was not modified during transmission.

⚡ Availability

Systems and data are accessible to authorized users when needed. DoS attacks target this pillar by making systems unavailable.

Non-repudiation — a 4th property often added. It means a sender cannot deny having sent a message. Digital signatures provide non-repudiation.
📝 Exam tip: Every attack targets one or more CIA pillars. In exam questions, always identify which pillar is being attacked — this shows understanding. DoS = Availability. Data theft = Confidentiality. File tampering = Integrity.
02

Threat Landscape — Malware

What is Malware?

Malware (Malicious Software) is any software intentionally designed to disrupt, damage, steal from, or gain unauthorized access to a computer system. It is the broadest category of cyber threats.

Types of Malware — Detailed

1. Virus

Attaches itself to a legitimate file or program. When the infected file is opened or shared, the virus executes and spreads. Requires human action to activate. Example: user opens an infected email attachment.

2. Worm

Self-replicating malware that spreads across networks without any human interaction. It exploits network vulnerabilities to move automatically from machine to machine. Spreads far faster than a virus. Example: WannaCry worm spread to 200,000+ computers in 24 hours with no user clicks.

3. Trojan Horse

Disguises itself as legitimate, useful software (games, tools, free apps). Once installed, it opens a backdoor for the attacker. Does NOT self-replicate. Victim is tricked into installing it willingly.

4. Ransomware

Encrypts the victim's files and demands a ransom (usually in Bitcoin/cryptocurrency) in exchange for the decryption key. Even after paying, there is no guarantee files will be restored. Attacks Confidentiality and Availability.

5. Spyware

Secretly monitors user activity — websites visited, files accessed, personal data — and sends it to the attacker without the user's knowledge. Often installed alongside free software.

6. Adware

Displays unwanted advertisements, often by redirecting browsers to ad-filled pages. Generally less harmful but degrades performance and can lead to worse malware. Often bundled with free downloads.

7. Rootkit

Embeds deep into the operating system to give an attacker persistent, administrator-level access. Designed to be invisible — very hard to detect even by antivirus software. Often used to hide other malware.

8. Keylogger

Records every keystroke typed by the user. Captures passwords, credit card numbers, personal messages, and bank details. Can be hardware (physical device on keyboard) or software.

9. Botnet

A network of computers infected with malware and secretly controlled by an attacker (called a bot-herder). These "zombie" machines are used to launch DDoS attacks, send spam, or mine cryptocurrency. Owners usually don't know their machine is infected.

How Malware Spreads

  • Phishing emails — malicious attachments or links in emails
  • Infected USB drives — plugging in an unknown USB device
  • Malicious websites — drive-by downloads (just visiting = infection)
  • Software vulnerabilities — unpatched OS or applications
  • Pirated software — free downloads bundled with malware
  • Network propagation — worms spread through open ports

Protection Against Malware

  • Install and regularly update antivirus / anti-malware software
  • Keep operating system and apps patched (patch management)
  • Never open suspicious email attachments
  • Regular offline backups (especially against ransomware)
  • Use firewalls and IPS to block malicious traffic
  • Avoid downloading pirated software
📝 Key difference exam question: Virus vs Worm → Virus needs human action; Worm spreads automatically. Trojan vs Virus → Trojan disguises as legit software, doesn't replicate. Rootkit → hides in OS, very hard to detect.

Phishing & Social Engineering

Phishing

Phishing is a cyberattack where an attacker sends fraudulent messages (typically emails) that appear to come from a trusted source — bank, government, company — to trick victims into revealing sensitive information (passwords, credit card numbers) or installing malware.

Phishing exploits human trust, not technical vulnerabilities. It is one of the most common attack vectors used today.

Types of Phishing

TypeDescriptionTarget
Phishing (mass)Bulk emails sent to thousands pretending to be banks, Amazon, PayPal etc. asking to "verify account"Anyone
Spear PhishingTargeted attack using personal information (name, employer, role) to appear more convincingSpecific individual
WhalingSpear phishing targeting high-value executives — CEO, CFO, board membersC-level executives
VishingVoice phishing — phone calls pretending to be bank or IT supportAnyone
SmishingSMS phishing — fake text messages with malicious linksMobile users
PharmingRedirects user from a real website to a fake one via DNS poisoning — no click requiredWeb users

Red Flags of Phishing

  • Urgency: "Your account will be suspended in 24 hours!"
  • Suspicious sender: support@amaz0n.com (note the zero)
  • Generic greeting: "Dear Customer" instead of your name
  • Suspicious links: URL doesn't match the real website
  • Requests for sensitive info: Banks never ask for passwords via email

Social Engineering

Social engineering is the manipulation of human psychology to trick people into revealing confidential information or performing security-compromising actions. It is called "hacking humans" because it bypasses technical defenses by exploiting people instead of systems.

Kevin Mitnick, one of history's most famous hackers, said social engineering was almost always easier than technical hacking.

Social Engineering Techniques

Pretexting

Creates a fake scenario to extract info. "I'm from IT — I need your password to fix a critical issue."

Baiting

Leaves infected USB drives in parking lots or offers free downloads. Exploits human curiosity.

Tailgating

Physically follows an authorized person into a restricted area. "Hold the door please!"

Quid Pro Quo

Offers a service in exchange for credentials. "I'll fix your slow PC for free — just give me remote access."

Impersonation

Pretends to be an authority figure — IT admin, auditor, manager — to gain trust and access.

Shoulder Surfing

Physically watches someone type their password or PIN in public places like cafes or ATMs.

Psychological Principles Exploited

Authority

"I'm calling from the CEO's office…"

Urgency

"Act NOW or lose your account!"

Fear

"Your computer has been hacked!"

Trust / Familiarity

Impersonates a known colleague

Defence Against Social Engineering

  • Security awareness training for all employees (most important defence)
  • Always verify identity before sharing any sensitive information
  • Never share passwords — even with "IT support"
  • Strict physical access control policies
  • Question unsolicited requests, even from known contacts
📝 Exam tip: Social engineering exploits humans, not systems. Key difference from hacking: no technical vulnerability needed. Phishing IS a form of social engineering (digital). Tailgating is physical social engineering.
04

Ransomware & Denial of Service (DoS/DDoS)

Ransomware — Deep Dive

Ransomware encrypts a victim's files, making them completely inaccessible, then demands payment (ransom) for the decryption key. It is one of the most financially damaging types of cybercrime.

How a Ransomware Attack Unfolds

1
Entry: Delivered via phishing email attachment, malicious link, or exploiting an unpatched vulnerability
2
Installation: Ransomware silently installs itself and establishes communication with attacker's C2 server
3
Encryption: Files (documents, images, databases, videos) are encrypted with the attacker's key
4
Ransom demand: A message appears on screen demanding Bitcoin payment within a deadline (e.g. 72 hours)
5
Outcome: Either pay ransom (no guarantee of recovery) or restore from clean backups
📌 CASE STUDY — WannaCry Ransomware (May 2017)
  • Exploited EternalBlue (CVE-2017-0144) — a vulnerability in Windows SMB protocol
  • Microsoft had released patch MS17-010, but many organizations hadn't applied it
  • Unlike normal ransomware, WannaCry behaved like a worm — spread automatically, no user clicks needed
  • 200,000+ computers infected across 150 countries within 24 hours
  • UK NHS severely disrupted — 80+ hospitals affected, surgeries cancelled, patient records locked
  • Researcher Marcus Hutchins found and registered a kill-switch domain, slowing the outbreak
  • Organizations with updated IPS signatures = zero infections. Without IPS = massive losses.
  • Lesson: Patch management + IPS updates = best ransomware defence

Denial of Service (DoS & DDoS)

A DoS attack floods a system, server, or network with so much traffic or so many requests that it becomes overwhelmed, slows down, or completely crashes — making it unavailable to legitimate users. DoS attacks target the Availability pillar of CIA.

DoS — Single Source
  • One attacker machine sends massive traffic to target
  • Easier to block — just block that single IP address
  • Less powerful than DDoS
DDoS — Distributed
  • Thousands of compromised machines (botnet) attack simultaneously
  • Very hard to stop — traffic comes from thousands of IPs
  • Much more powerful and damaging

Types of DoS Attacks

TypeHow it worksExample
Volume-basedFloods bandwidth with massive data (UDP/ICMP floods)UDP flood, ICMP flood
SYN FloodSends many TCP SYN requests but never completes handshake — fills server's connection tableProtocol attack
HTTP FloodSends massive legitimate-looking HTTP requests to exhaust web server resourcesApplication layer attack
Smurf AttackSends ICMP requests with victim's spoofed IP to broadcast — all devices reply to victimAmplification attack
Ping of DeathSends oversized malformed ping packets that crash the target systemOlder attack, rare today

Protection Against DoS/DDoS

  • Rate limiting — restrict number of requests from a single IP
  • Firewalls & IPS — detect and block flood patterns
  • CDN / traffic scrubbing — services like Cloudflare absorb and filter attack traffic
  • Blackhole routing — route all attack traffic to null (dropped)
  • Anycast network diffusion — spread traffic across multiple servers
📝 Exam tip: DoS = 1 attacker. DDoS = botnet (many attackers). Both target Availability. SYN flood = half-open connections fill up server table. DDoS is harder to stop because blocking one IP doesn't help.

Firewalls

What is a Firewall?

A firewall is a network security system that monitors and controls incoming and outgoing network traffic based on predetermined security rules. It acts as a barrier between a trusted internal network and untrusted external networks (like the Internet).

Analogy: Like the firewall in a car that sits between the engine and passenger compartment — it protects passengers from engine fires. Similarly, a network firewall protects internal systems from external threats.

Types of Firewalls

Host-based Firewall

Installed on an individual device (server, workstation). Monitors traffic going in and out of that single machine via the NIC. Example: Windows Defender Firewall built into Windows OS. Organizations often combine host-based + network firewalls for "defense-in-depth."

Network-based Firewall

A hardware appliance (or virtual appliance) that protects an entire network. Has 2+ NICs and all network traffic passes through it. Placed at the network border between the internal network (intranet) and the Internet. Filters traffic based on firewall rules.

Web Application Firewall (WAF)

Specifically protects web applications. Sits between the web server and its clients. Protects against web-specific attacks like Cross-Site Scripting (XSS), SQL Injection, and session hijacking. Can be a standalone appliance or software module.

Firewall Generations / Rule Types

Stateless (1st Gen)

Uses Access Control Lists (ACLs). Treats each packet as a new independent event. No memory of past traffic. Fast but less secure. Similar to how a router filters traffic.

Stateful (2nd Gen)

Tracks session state. Knows whether a packet is part of an established session or a new suspicious connection. More intelligent — blocks traffic that isn't part of a valid session.

NGFW (Next-Gen)

Adds deep packet inspection (DPI) on top of stateful inspection. Can inspect application-layer content, detect malware within traffic, and apply application-aware rules. Most advanced.

Sample Firewall Rules Table

Rule #Source IPDestinationPortProtocolAction

📝 Exam tip: Stateless = no memory, each packet treated fresh. Stateful = tracks sessions, smarter. NGFW = deep packet inspection + application awareness. WAF = protects web apps specifically.

Intrusion Detection System (IDS) & Intrusion Prevention System (IPS)

Core Difference — IDS vs IPS

IDS — Intrusion Detection System
  • Monitors network traffic
  • Detects suspicious activity
  • Sends ALERTS only — does NOT block
  • Reactive — responds after the fact
  • Operates in monitoring mode
  • Placed after the firewall for traffic analysis
IPS — Intrusion Prevention System
  • Monitors network traffic
  • Detects suspicious activity
  • BLOCKS attacks in real time
  • Proactive — stops threats before damage
  • Operates inline (traffic flows through it)
  • Placed after firewall, before internal network

Both use the same detection methods. The key difference is the response: IDS alerts, IPS blocks.

Types of IDS

1. NIDS — Network Intrusion Detection System

Deployed at strategic network points to monitor traffic from all devices across subnets. Compares traffic against known attack signatures. When an attack is detected, alerts the administrator. Example: NIDS placed in front of firewalls to catch firewall attacks.

2. HIDS — Host Intrusion Detection System

Operates on individual devices (hosts). Monitors traffic and file integrity on that specific machine. Compares current system files with previous snapshots to detect unauthorized changes. Used on critical servers.

3. Hybrid IDS

Combines NIDS + HIDS for comprehensive protection. Gives a complete view of both network and host activity. Example: Prelude IDS.

4. APIDS — Application Protocol-based IDS

Monitors and analyzes application-specific protocols to detect attacks targeting specific applications. Example: monitoring SQL queries to a web server's database for malicious activity.

5. PIDS — Protocol-based IDS

Monitors communication protocols between devices (HTTP, HTTPS) to detect abnormal activity patterns. Example: monitoring HTTPS traffic entering a web server.

IDS/IPS Comparison Table

ParameterFirewallIPSIDS
PurposeFilters traffic by rulesDetects & prevents in real-timeDetects & alerts
ResponseBlock or permit by ruleActively blocks threatsGenerates alerts
ModeInline at network boundaryInline, after firewallMonitoring mode (passive)
Traffic pathPrimary route of trafficAfter firewallAfter IPS, for analysis
BehaviourStateless / Stateful rulesProactive preventionReactive detection
📌 CASE STUDY — WannaCry & IPS (2017)
  • WannaCry exploited EternalBlue (SMB vulnerability) to spread as a worm
  • IPS devices with updated signatures recognized the malicious SMB traffic pattern immediately
  • As soon as attack packets were detected, IPS blocked them — zero infections in protected organizations
  • Organizations without IPS or without updated signatures suffered massive losses
  • Proves: IPS provides proactive security. Prevention > Detection.
  • Also shows importance of regularly updating IPS signatures (like antivirus definitions)
📝 Exam tip: IDS = detect + alert (passive/reactive). IPS = detect + block (inline/proactive). Both use same detection methods. BIGGEST difference = response to attack. IPS must be inline for traffic to pass through it.

Cryptography — Symmetric & Asymmetric Encryption

What is Cryptography?

Cryptography (literally "secret writing") is the art of securing information by encoding it so that only authorized parties can read it. Unlike "security through obscurity" (hiding something and hoping no one finds it), cryptography works even if the attacker knows the method — they still cannot read the data without the key.

Key Terminology

Plaintext / Cleartext

Original unencrypted message that anyone can read

Ciphertext

The scrambled, encrypted message — unreadable without the key

Cipher

The algorithm (process) used to encrypt and decrypt a message

Cryptanalysis

The art of breaking or cracking cryptographic systems

Standard Characters in Crypto Explanations

  • Alice — the sender of the message
  • Bob — the intended recipient
  • Mallory — a malicious attacker trying to intercept

Symmetric Encryption

In symmetric encryption, the same secret key is used for both encryption and decryption. Both sender and receiver must have the same key, which must be kept secret at all times.

How it works (step by step)

1
Alice and Bob meet (physically or via secure channel) and agree on a cipher and secret key value. Both store this key securely.
2
Alice encrypts her file using the cipher and the shared secret key → produces ciphertext
3
Alice sends only the ciphertext to Bob over the network
4
Bob receives the ciphertext and decrypts it using the same shared secret key → gets original plaintext
Advantages
  • Very fast encryption and decryption
  • Ideal for bulk encryption of large data
  • Low computational overhead
Disadvantages
  • Key distribution problem — how to securely share the key?
  • If key is intercepted, security is completely broken
  • Doesn't scale well — every pair needs a unique key

Symmetric Encryption Algorithms

🔐
AES (Advanced Encryption Standard) — current gold standard, very secure, used everywhere (WiFi, TLS, file encryption)
🔐
DES (Data Encryption Standard) — old 56-bit key, now considered insecure (brute-forceable)
🔐
3DES (Triple DES) — applies DES three times for better security, still slower than AES
🔐
Blowfish / Twofish — fast and secure alternatives to DES
🔐
RC4 — stream cipher, used in older WEP WiFi (now insecure)
🔐
ChaCha20 — modern fast stream cipher used in TLS 1.3

Asymmetric Encryption (Public Key Cryptography)

Asymmetric encryption uses a pair of mathematically linked keys — a public key and a private key. What one key encrypts, only the other key can decrypt. The public key is shared openly; the private key is kept secret by its owner.

How it works (step by step)

1
Bob generates a key pair — a public key and a private key. He keeps the private key secret.
2
Bob publishes his public key — posts it publicly so anyone can access it. Alice downloads Bob's public key.
3
Alice encrypts her message using Bob's public key → produces ciphertext
4
Alice sends ciphertext to Bob over the network
5
Bob decrypts the ciphertext using his private key → gets Alice's original message
6
Mallory can intercept both the message and the public key, but cannot decrypt the message — the public key cannot reverse the encryption. Only Bob's private key can.
Advantages
  • No need to share a secret key beforehand
  • Secure over public networks
  • Provides authentication (identity proof)
  • Enables digital signatures
Disadvantages
  • Much slower than symmetric encryption
  • High computational overhead
  • Message size limited by key size
  • Not suitable for bulk data encryption

Use Cases of Asymmetric Encryption

  • Authentication — proving identity (the private key holder cannot be impersonated)
  • Key exchange — securely agreeing on a symmetric key (then use symmetric for bulk data)
  • Digital signatures — proving a message was sent by a specific person
  • HTTPS/TLS — asymmetric used at the start to exchange a symmetric session key

Asymmetric Encryption Algorithms

RSA

Most widely used. Based on difficulty of factoring large numbers. Used in HTTPS, digital signatures.

Diffie-Hellman (DH)

Key exchange protocol. Allows two parties to agree on a shared secret over a public channel.

ECC

Elliptic Curve Cryptography. Smaller key sizes for same security level. Used in mobile & IoT.

DSA / ECDSA

Digital Signature Algorithm. Used for creating and verifying digital signatures.

📝 Exam tip: Symmetric = same key, fast, bulk data, key sharing problem. Asymmetric = public+private key pair, slow, authentication & key exchange. In HTTPS: asymmetric used first to exchange a symmetric session key, then symmetric used for actual data.

Hashing & Digital Signatures

Cryptographic Hashing

A hash function takes any input (text, file, password, message — any size) and produces a fixed-length output called a hash value or digest. Regardless of whether the input is 1 byte or 10 GB, the output length is always the same.

Key Properties of Hashing

One-way function

Input → Hash (easy). Hash → Input (impossible). You cannot reverse a hash to get the original data.

Deterministic

Same input always produces the exact same hash. This is how integrity checking works.

Avalanche Effect

Changing even 1 character in the input produces a completely different hash. "hello" vs "Hello" → totally different SHA-256 outputs.

Collision resistant

Two different inputs should never produce the same hash (weak algorithms like MD5 fail this).

Uses of Hashing

  • Password storage — systems store the hash of the password, not the password itself. When you log in, your typed password is hashed and compared.
  • File integrity checking — download a file, hash it, compare with published hash to verify it wasn't tampered with.
  • Digital signatures — hash a message, then sign the hash.
  • Blockchain — each block contains the hash of the previous block, creating an unbreakable chain.

Hashing Algorithms — Secure vs Insecure

AlgorithmStatusNotes
MD5INSECUREFast — billions of hashes/sec. Vulnerable to rainbow tables & collisions. Never use for passwords.
SHA-1INSECURECollision attacks demonstrated. Deprecated by major browsers.
SHA-256SECUREPart of SHA-2 family. Widely used for file integrity and certificates.
SHA-3SECURENewest standard. Different design from SHA-2. Highly secure.
bcryptBEST FOR PASSWORDSDeliberately slow + adds unique salt. Makes brute-force attacks impractical.
scrypt / Argon2SECUREMemory-hard algorithms. Even harder to brute-force than bcrypt.

What is a Salt?

A salt is a random unique value added to a password before hashing. Even if two users have the same password, their salted hashes will be completely different, defeating rainbow table attacks.

📌 CASE STUDY — LinkedIn Data Breach (June 2012)
  • Hackers stole approximately 6.5 million user password hashes
  • LinkedIn had stored passwords using unsalted MD5 — considered insecure even then
  • MD5 can generate billions of hashes per second → easy to brute-force
  • No unique salt → users with same password had identical hashes → attackers could identify common passwords instantly
  • Attackers used rainbow tables (precomputed hash databases) — cracked nearly 90% of passwords within 24 hours
  • Should have used: bcrypt with unique salts — slow by design, each hash unique
  • Later in 2016, nearly 117 million accounts went on sale online
  • Lesson: Never use MD5 or SHA-1 for passwords. Always use bcrypt/Argon2 with salting.

Digital Signatures

A digital signature combines hashing and asymmetric encryption to achieve two things simultaneously: prove the sender's identity (authentication) and verify the message wasn't tampered with (integrity). It also provides non-repudiation — the sender cannot deny having sent the message.

How Digital Signatures Work (step by step)

1
Alice creates a hash of her message using a pre-agreed algorithm (e.g., SHA-256)
2
Alice encrypts the hash with her own private key → this is the digital signature
3
Alice sends both the original message AND the digital signature to Bob
4
Bob decrypts the signature using Alice's public key → reveals Alice's original hash
5
Bob independently hashes the received message using the same algorithm
6
Bob compares the two hashes — if they match: message is untampered and definitely from Alice ✅. If they don't match: message was altered or someone impersonated Alice ❌
📝 Exam tip: Hash = one-way, fixed output, ensures integrity. Digital signature = hash + private key encryption = proves identity + integrity + non-repudiation. Alice signs with private key, Bob verifies with Alice's public key.

Cybersecurity Tools

Cybersecurity tools form a complete security lifecycle: Detection (monitoring) → Analysis (scanning) → Prevention and response (frameworks). They don't work in isolation.

1. Network Monitoring Tools — Understanding Traffic

Wireshark Open Source

An open-source packet analyzer that captures and inspects every data packet moving across a network in real time. Security professionals can see exactly what data is being transmitted — including protocols, source/destination IPs, and even unencrypted passwords.

  • Troubleshoot network issues (DNS failures, packet drops)
  • Analyze protocols: HTTP, TCP, DNS, FTP
  • Detect suspicious activity (e.g., unencrypted passwords over HTTP)
  • Digital forensics — identify C2 (command-and-control) communication from malware
  • Limitation: Requires technical expertise and admin privileges. Complex to interpret.

2. Network Scanning Tools — Discovering Attack Surface

Nmap Open Source

Network Mapper — discovers active devices, open ports, running services, and OS on a network. Critical first step in both ethical hacking and penetration testing.

  • Discover all active hosts in a network
  • Identify open ports (potential attack entry points)
  • Determine running services and versions
  • Detect operating systems

Port status meaning: Open port = available service (potential entry point). Closed port = inactive. Filtered port = firewall is blocking it.

Real-world use: Penetration testers use Nmap to map the entire network before simulating attacks. Attackers also use it to find vulnerable systems before ransomware deployment.

3. Vulnerability Assessment Tools — Finding Weaknesses

Nessus Commercial
  • By Tenable. Enterprise-grade scanner
  • Scans for misconfigs, outdated software, known CVEs
  • Classifies: Critical / High / Medium / Low
  • Accurate results, helps prioritize fixes
  • Costly and needs proper configuration
OpenVAS Free
  • Free and open-source alternative
  • Comprehensive network scans
  • Good for education, small orgs, research
  • Steeper learning curve
  • Slower performance vs commercial tools
☁️
Qualys VMDR — Cloud-based platform. Real-time threat intelligence. Suitable for large organizations. Includes patching capabilities.
🎯
Rapid7 InsightVM — Focuses on risk prioritization using real-world threat intelligence. Helps fix the most critical vulnerabilities first.
📌 CASE STUDY — Equifax Data Breach (2017)
  • Apache Struts vulnerability CVE-2017-5638 — CVSS score 10.0 (maximum severity)
  • Publicly disclosed and patched months before the breach — Equifax simply didn't apply it
  • A basic Nessus or OpenVAS scan would have flagged this vulnerability immediately
  • Attackers exploited it → accessed 147 million people's names, SSNs, financial details
  • Settlement: ~$575 million. Multiple executives faced legal consequences
  • Lesson: Regular vulnerability scanning + immediate patching is non-negotiable

4. SIEM & Log Monitoring — Detecting Attacks in Real Time

SIEM (Security Information and Event Management) tools collect, aggregate, and analyze log data from all systems across the organization to detect suspicious patterns and anomalies.

Splunk — Leading commercial SIEM. Analyzes massive volumes of machine data. Identifies unusual patterns: multiple failed login attempts, unauthorized access, abnormal data transfers. Expensive and complex but very powerful.
ELK Stack (Elasticsearch + Logstash + Kibana) — Free, open-source log aggregation and visualization stack. Widely used in enterprises for log management and security monitoring.
Nagios / Zabbix — Infrastructure monitoring tools. Track system performance, uptime, and availability. Generate alerts when thresholds are exceeded.
📌 CASE STUDY — British Airways Data Breach (2018)
  • Attackers injected malicious JavaScript into British Airways' website
  • Script silently captured customer payment card details and personal data
  • ~400,000 customer records compromised
  • Breach was NOT detected immediately — shows lack of effective SIEM monitoring
  • Under GDPR, organizations must detect and report breaches within 72 hours
  • ICO (UK regulator) fined British Airways £20 million under GDPR
  • Lesson: Real-time SIEM monitoring would have detected the unusual data exfiltration. Continuous monitoring is critical.

5. Penetration Testing Tools — Simulating Attacks

Metasploit Framework — The most widely used penetration testing tool. Allows ethical hackers to simulate real-world attacks on systems to find exploitable vulnerabilities before malicious attackers do. Contains a huge library of known exploits. Used only by authorized security professionals and researchers.

Penetration testing (pen testing) is the authorized practice of attacking a system to find its weaknesses. Companies hire ethical hackers to test their systems using tools like Metasploit.

📝 Exam tip: Wireshark = packet capture/analysis. Nmap = network/port scanning. Nessus/OpenVAS = vulnerability assessment. Splunk/ELK = SIEM (log monitoring). Metasploit = penetration testing (ethical hacking). Know which tool does what!

Cyber Laws & Data Protection Regulations

Cyber laws make it mandatory for organizations to protect personal and sensitive data, inform users when breaches occur, and be transparent about data usage. Non-compliance leads to heavy fines, legal actions, and reputational damage.

Major Regulations — Detailed

GDPR — General Data Protection Regulation European Union

The world's most powerful and wide-reaching data protection law. Applies not just to EU companies, but to ANY organization globally that processes data of EU citizens. Key features:

  • Right to be Forgotten — users can request complete deletion of their personal data
  • Data Portability — users can request their data in a portable format
  • 72-hour breach reporting — organizations must report breaches to regulators within 72 hours
  • Consent-based data collection — explicit consent required before collecting data
  • Penalties: up to €20 million or 4% of global annual revenue (whichever is higher)

Case: British Airways fined £20M. Google fined €50M for lack of transparency.

HIPAA — Health Insurance Portability and Accountability Act USA — Healthcare

Protects medical and health-related information. Applies to hospitals, clinics, health insurance companies, and employers who handle patient data. Requires strong encryption, access controls, and audit trails for all health records.

Case: Anthem Healthcare Breach (2015) — 78 million health records exposed. $16 million HIPAA penalty.

PCI DSS — Payment Card Industry Data Security Standard Global — Payments

Not a law but a mandatory compliance requirement for any organization that handles, processes, or stores credit/debit card data. Defines 12 security requirements including encryption, access control, network monitoring, regular security testing, and vulnerability management.

Case: Target Data Breach (2013) — 40 million cards stolen. Weak PCI DSS controls. $200 million in losses.

GLBA — Gramm-Leach-Bliley Act USA — Finance

Applies to financial institutions (banks, insurance companies, investment firms). Requires them to: inform customers about what financial data is collected and how it's used; protect that data; give customers the right to opt out of data sharing with third parties. Emphasizes transparency.

CCPA — California Consumer Privacy Act California, USA

State-level privacy law giving California residents strong rights over their personal data:

  • Right to know what personal data is collected about them
  • Right to delete their personal data
  • Right to opt-out of the sale of their personal data
  • Right to non-discrimination for exercising these rights

Compliance Frameworks & Evidence Handling

Compliance Frameworks

Frameworks provide structured guidelines that help organizations implement security controls systematically. They bridge the gap between technology and legal/regulatory requirements.

ISO 27001

International standard for Information Security Management Systems (ISMS). Provides a framework for managing security risks systematically.

ISO 27701

Extension of ISO 27001 for Privacy Information Management. Helps organizations comply with GDPR and other privacy regulations.

NIST CSF

National Institute of Standards & Technology Cybersecurity Framework. Five functions: Identify → Protect → Detect → Respond → Recover

PCI DSS

12 security requirements for payment card data. Mandatory for all card-processing organizations globally.

Compliance Monitoring

Internal Monitoring

Regular audits conducted by the organization's own team to verify that policies and standards are being followed. Example: a bank's internal audit team checks quarterly whether customer data is encrypted and access is properly restricted.

External Monitoring

Performed by third-party auditors or regulatory bodies to provide an unbiased, independent assessment of compliance. Example: companies handling payment data must undergo annual PCI DSS audits by qualified security assessors.

Consequences of Non-Compliance

  • Heavy financial fines (GDPR: up to 4% of global revenue)
  • Legal actions and lawsuits from affected individuals
  • Loss of business licenses (e.g., losing ability to process payments)
  • Severe reputational damage and loss of customer trust
  • Long-term financial losses from customer churn

Legal Concepts — Digital Evidence Handling

In cybersecurity investigations, legal processes are critical. Digital evidence must be handled correctly or it becomes inadmissible in court.

Legal Hold

Preserving all potentially relevant data when litigation is anticipated. Data cannot be deleted or modified.

eDiscovery

Process of identifying, collecting, and preserving electronic information for use as evidence in legal proceedings.

Chain of Custody

A documented trail showing who accessed, handled, or transferred evidence. If broken, evidence becomes invalid in court.

📝 Exam tip: NIST CSF = Identify, Protect, Detect, Respond, Recover (memorize this order). Chain of custody = evidence integrity. If evidence is tampered with = inadmissible in court. Internal audit = own team. External audit = third party.

All Case Studies — Quick Revision

Every case study follows the same pattern: What happened → Why it happened → Impact → What should have been done → Lesson.

📌 WannaCry Ransomware (May 2017)
  • What: Ransomware worm exploiting Windows SMB vulnerability (EternalBlue / CVE-2017-0144)
  • Why: Organizations hadn't applied Microsoft patch MS17-010
  • Impact: 200,000+ computers, 150 countries, 24 hours. NHS UK paralysed — surgeries cancelled
  • Stopped by: Marcus Hutchins registering kill-switch domain
  • Lesson: Patch management + updated IPS signatures = complete protection. Prevention > Detection.
  • Concept: IPS, Ransomware, Patch Management
📌 LinkedIn Data Breach (June 2012)
  • What: 6.5 million password hashes stolen from LinkedIn's database
  • Why: LinkedIn used unsalted MD5 hashing — insecure even at the time
  • Impact: 90% of passwords cracked within 24 hours using rainbow tables. 117M accounts leaked in 2016.
  • Should have used: bcrypt with unique salts per password
  • Lesson: Never use MD5/SHA-1 for passwords. Always use slow hashing (bcrypt/Argon2) with salts.
  • Concept: Hashing, Password Storage, Rainbow Tables, Salting
📌 Equifax Data Breach (2017)
  • What: Apache Struts vulnerability CVE-2017-5638 (CVSS 10.0) exploited
  • Why: Publicly known vulnerability left unpatched for months — lack of due diligence
  • Impact: 147 million people's names, SSNs, financial details exposed. $575M settlement.
  • Should have used: Regular Nessus/OpenVAS scans would have detected the unpatched vulnerability
  • Lesson: Vulnerability scanning must be continuous. Known vulnerabilities must be patched immediately.
  • Concept: Vulnerability Assessment, Patch Management, Nessus, OpenVAS
📌 British Airways Data Breach (2018)
  • What: Malicious JavaScript injected into BA's website captured customer payment and personal data
  • Why: No effective real-time SIEM monitoring in place
  • Impact: ~400,000 customer records. ICO fined BA £20 million under GDPR.
  • Should have used: SIEM (Splunk/ELK) monitoring to detect unusual data exfiltration in real time
  • Lesson: Continuous monitoring + GDPR 72-hour reporting requirement. Early detection saves millions.
  • Concept: SIEM, GDPR, Continuous Monitoring, Compliance
📌 Anthem Healthcare Breach (2015)
  • What: Hackers accessed personal and medical information of 78 million people
  • Why: Insufficient security controls protecting health records
  • Impact: $16 million HIPAA penalty — largest at the time
  • Lesson: Health data requires maximum protection. HIPAA compliance is mandatory.
  • Concept: HIPAA, Data Protection, Healthcare Security
📌 Target Data Breach (2013)
  • What: Attackers stole credit card data of 40 million customers via compromised HVAC vendor credentials
  • Why: Weak third-party vendor access controls, failure to comply with PCI DSS
  • Impact: $200+ million in losses, massive reputational damage
  • Lesson: PCI DSS compliance is non-negotiable. Third-party access must be strictly controlled.
  • Concept: PCI DSS, Access Control, Third-party Risk

Master Summary — One Line Per Topic

🔐
CIA Triad: Confidentiality (who can access), Integrity (accuracy), Availability (accessible when needed)
🦠
Virus vs Worm: Virus needs human action. Worm self-propagates. Trojan disguises as legit software.
🎣
Phishing: Mass fake emails. Spear = targeted. Whaling = CEO. Vishing = voice. Smishing = SMS.
💥
DoS vs DDoS: DoS = 1 attacker. DDoS = botnet. Both kill Availability. SYN flood fills connection table.
🔥
Firewalls: Stateless = no memory. Stateful = tracks sessions. NGFW = deep packet inspection. WAF = web apps.
🛡️
IDS vs IPS: IDS = detect + alert (passive). IPS = detect + block (active/proactive). Same methods, different response.
🔑
Symmetric: Same key both sides. Fast, bulk data. Problem: key sharing. Algorithms: AES, DES, 3DES.
🗝️
Asymmetric: Public+private key pair. Slow. Authentication & key exchange. Algorithms: RSA, ECC, DH.
#️⃣
Hashing: One-way, fixed output, same input = same hash. Avalanche effect. bcrypt best for passwords.
✍️
Digital Signature: Hash + private key encryption. Proves identity + integrity + non-repudiation.
🛠️
Tools: Wireshark=packets. Nmap=ports. Nessus/OpenVAS=vulns. Splunk=SIEM. Metasploit=pen testing.
⚖️
Laws: GDPR=EU/global/72hr. HIPAA=health. PCI DSS=payments. GLBA=finance. CCPA=California.
